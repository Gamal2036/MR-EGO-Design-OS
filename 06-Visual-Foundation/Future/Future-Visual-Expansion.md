# Future Visual Expansion

**Phase:** DP-5 (Visual Foundation)
**Version:** 1.0
**Status:** FOUNDATION — Permanent
**Design Authority:** DP-0 ([Future-Expansion.md](../../01-Constitution/Future-Expansion.md))

---

## Definition

Future Visual Expansion defines how the Visual Foundation grows with the platform. As new modules, themes, and capabilities are added, the visual language must expand without breaking existing design coherence.

---

## What Cannot Change

### Permanent Visual Identity

| Element | Permanent | Why |
|---------|-----------|-----|
| Visual Personality | Confident, Precise, Warm, Restrained | Constitutional brand identity |
| Visual Emotion | Calm competence, Quiet ambition, Warm authority | Core to brand feeling |
| Primary Color | Blue (#2563EB / #3B82F6) | Brand identity |
| Light Source | Top-left at −45° | Visual physics model |
| Depth System | 6 active levels (0–5), 4 reserved (6–9) | Architecture constraint |
| Glass Model | Frosted blur, no shadow, no text on glass | Material behavior |
| Motion Purpose | Every animation communicates meaning | Motion philosophy |
| Restraint Principle | Less is more across all visual dimensions | Core brand value |

## What Can Expand

### Themes

| Theme | Target | Visual Changes |
|-------|--------|---------------|
| High Contrast | Users with visual impairments | Max contrast ratios, thicker borders, bolder focus rings |
| OLED | OLED device users | Pure black backgrounds, reduced power consumption |
| Warm | Personal preference | Warmer neutrals, amber-tinted grays |
| Cool | Personal preference | Cooler neutrals, blue-tinted grays |
| Module-specific | Per-module branding | Module accent color in headers, icons, badges |

### Visual Patterns

| Pattern | Future Possibility | Constraint |
|---------|-------------------|------------|
| AI visual states | More nuanced AI expression | Must use Primary color family |
| Data visualization | Rich interactive data views | Must maintain accessibility |
| Illustrations | New illustration categories | Must follow existing style |
| Icon set | Additional icons for new features | Must follow existing grid/stroke |
| Animation | New transition patterns | Must be purposeful, respectful |

### Module Visual Identity

| Module | Visual Distinctiveness | Shared Foundation |
|--------|----------------------|-------------------|
| Career | Green accent, growth motifs | Core design tokens |
| Learning | Teal accent, discovery motifs | Core design tokens |
| Projects | Indigo accent, structure motifs | Core design tokens |
| CRM | Violet accent, connection motifs | Core design tokens |
| Documents | Slate accent, focus motifs | Core design tokens |

Each module inherits the full Visual Foundation and adds only:
- Accent color for module-level UI
- Module-specific illustration style
- Module-specific icon treatments
- Module-specific empty/success states

## Expansion Governance

### Visual Review Gate

Every visual expansion must pass:

1. **Personality fit** — Does it feel Confident, Precise, Warm, Restrained?
2. **Emotion alignment** — Does it produce Calm competence?
3. **Constitutional alignment** — Does it violate any DP-0 rule?
4. **Design language consistency** — Does it match DP-1 tokens?
5. **Accessibility pass** — Does it meet WCAG AA?
6. **Performance budget** — Does it maintain 60fps?
7. **Backward compatibility** — Does it work in existing themes?
8. **Documentation completeness** — Is it fully documented with before/after?

### Visual Debt Prohibition

New visual features must not create visual debt:
- No visual styles that are "temporary" or "will be fixed later"
- No inconsistent patterns that will need migration
- No theme-specific behaviors without cross-theme parity
- No "light mode only" patterns — dark mode parity required for launch

---

## Visual Scalability

The Visual Foundation supports unlimited expansion because:

1. **Visual decisions are principles, not prescriptions.** The "why" is documented, enabling consistent expansion.
2. **Design tokens are extensible.** New tokens are additive, not transformative.
3. **Accent colors are additive.** Module accent colors sit beside, not on top of, the core palette.
4. **Depth levels are reserved.** Levels 6–9 accommodate future overlay needs.
5. **Motion patterns are compositional.** New animations compose from existing duration/easing tokens.
6. **Glass properties are adjustable.** Within defined ranges, glass adapts without breaking.
