**Phase:** DP-5 (Visual Foundation)
**Version:** 1.0
**Status:** FOUNDATION — Permanent
**Design Authority:** DP-1 ([Illustration-Guidelines.md](../../02-Design-Language/Illustration-Guidelines.md))

---

## Definition

The Visual Foundation Illustrations document defines the emotional role of illustrations in MR:EGO. Illustrations are rare, purposeful, and always support the user's understanding — never decoration.

---

## Illustration Personality

MR:EGO illustrations are:

**Simple.** Geometric shapes, minimal detail. One concept per illustration.

**Warm.** Abstract human figures (when present) have gentle proportions and approachable posture.

**Purposeful.** Every illustration communicates a specific concept: empty state, success, onboarding, or error.

**Brand-aligned.** Illustrations use Primary + Neutral palette colors. No extraneous hues.

**Timeless.** Flat, geometric style — no trend-driven aesthetics like isometric, 3D, or detailed rendering.

---

## Emotional Quality by Type

### Empty State Illustrations

| Emotional Goal | How It Is Achieved |
|---------------|-------------------|
| Encouragement | Gentle composition, upward posture, inviting colors |
| Clarity | Clear concept (empty folder, blank page, no results) |
| Guidance | Directional cues (arrow, pointing, action suggestion) |
| Calm | Soft lines, generous negative space, no urgency |

### Success State Illustrations

| Emotional Goal | How It Is Achieved |
|---------------|-------------------|
| Satisfaction | Complete shape (circle, checkmark), centered composition |
| Closure | Final state — nothing more to do |
| Measured celebration | Subtle accent, no confetti, no burst effects |

### Onboarding Illustrations

| Emotional Goal | How It Is Achieved |
|---------------|-------------------|
| Welcome | Open composition, inviting colors, warm tones |
| Clarity | Step-by-step visual that matches the UI |
| Anticipation | Forward direction, progress indication |

### Error State Illustrations

| Emotional Goal | How It Is Achieved |
|---------------|-------------------|
| Reassurance | Broken element but not catastrophic |
| Clarity | Clear visual of what went wrong |
| Solution focused | Visual directs attention to fix action |

## Character Usage

| Aspect | Specification |
|--------|---------------|
| Human figures | Abstract/geometric only |
| Facial features | None — abstract head shapes only |
| Skin colors | None — use neutral palette colors |
| Gender | Neutral by default |
| Posture | Open, approachable, never aggressive |
| Objects held | Simple geometric shapes |
| Figures per illustration | Maximum 2 |
| Figure size | 40-60% of illustration area |
| Context | Abstract (no detailed backgrounds) |

---

## Abstract Graphics

When illustrations do not include human figures:

| Type | Emotional Quality | Examples |
|------|------------------|----------|
| Objects | Clear, recognizable | Document, folder, chart |
| Shapes | Calm, orderly | Circles, rounded rectangles, lines |
| Compositions | Balanced, intentional | Symmetrical or golden ratio |
| Negative space | Breathing, comfortable | 30-50% empty space |

---

## Illustration Rules

1. **Maximum 3 colors per illustration.** Fewer is better.
2. **Maximum size:** 240px inline, 400px full-page.
3. **Centered composition** within container.
4. **Transparent background** — never use background shapes.
5. **SVG format only** — responsive by nature.
6. **No gradients** — flat colors only.
7. **No shadows** — flat as the design language.
8. **No animation** — illustrations are static.
9. **Text never overlaps illustration** — separate layout elements.
10. **Scale down on mobile** — 120px width for small screens.

---

## AI Illustrations

AI in MR:EGO does not get its own abstract illustration character or mascot. AI is represented through:
- UI components (badges, glows, thinking indicators)
- Patterns (accent bars, subtle gradients)
- Iconography (sparkle/lightburst icon for AI)

No robot illustrations. No brain illustrations. No glowing orb illustrations. AI visual language is functional UI, not illustration.

---

## Illustration Anti-Patterns

| Anti-Pattern | Why |
|-------------|-----|
| Photography | Dates quickly, hard to maintain consistently |
| 3D renders | Fights flat design language |
| Isometric | Adds cognitive load, trend-driven |
| Brand mascot | Unprofessional for enterprise tool |
| Animated illustrations | Distracting, reduces calm |
| Data-driven illustrations | Charts serve this purpose |
| Detailed backgrounds | Competing with content focus |
| Realistic human figures | Representation issues, complex to maintain |
